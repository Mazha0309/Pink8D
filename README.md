# ⚙️ Pink8D (P8D) v3.0: 工业级活塞逻辑语言

**Pink8D** 是一门基于形态学（Morphological）设计的硬核底层语言。它抛弃了抽象的变量名，将计算逻辑映射为一套严密的液压推进系统。通过活塞冲程、腔体开闭以及管道流体交互，实现高效的逻辑处理。

---

## 🏗️ 核心形态学指令集

在 P8D 中，指令遵循“底座 (8) + 芯 (等号/点号) + 封头 (D/>)”的物理结构。**波浪线 `~` 的位置决定了流体交互的时序。**

### 1. 压力与位移 (数值与指针)

| 动作形态 | 物理意义 | 详细说明 |
| :--- | :--- | :--- |
| **`8====D`** | **封闭加压** | 增加当前格数值（`=` 数量即增加值） |
| **`8....D`** | **封闭减压** | 减少当前格数值（`.` 数量即减少值） |
| **`8====>`** | **右向推进** | 指针右移（等号数 + 1） |
| **`<====8`** | **左向回溯** | 指针左移（等号数 + 1） |

### 2. 流体交互 (I/O 榨取与灌注)

| 动作形态 | 物理意义 | 描述 |
| :--- | :--- | :--- |
| **`~8D`** | **先行吸入** | 先从 STDIN 读取 1 字节，再封闭活塞 |
| **`8D~`** | **后续喷射** | 先封闭活塞，再向 STDOUT 压入 1 字节 |
| **`~~8D`** | **强力抽吸** | 从当前对准的 **Hatch (舱门)** 文件读取 1 字节 |
| **`8D~~`** | **强力灌注** | 向当前对准的 **Hatch (舱门)** 文件写入 1 字节 |

### 3. 逻辑控制 (空腔与弹射)

* **`8{ ... }D`** : **逻辑空腔**。基于当前内存格数值的循环结构。
* **`8*D`** : **动态弹射**。指令指针立即跳转到当前内存格存储的地址（自举核心）。

## 🚀 v3.0 强力协议：舱门与铭牌

### 1. 机体铭牌 (The Machine Plate)

在文件顶部静态声明外部轨道咬合关系：

```text
    8[H0:input.8d]D    // 0号舱门：通常为输入轨
    8[H1:output.c]D    // 1号舱门：通常为输出轨
    ---
```

### 2. 舱门对准 (Hatch Alignment)

使用方括号活塞切换当前操作的管道轨道：

* **`8[]D`** : 旋转至 0 号舱门（默认输入）。
* **`8[=]D`** : 旋转至 1 号舱门（默认输出）。

## 🛠️ 工业组件库 (Toolchain)

Pink8D v3.0 提供全套开发工具链，满足不同场景的需求：

| 组件 | 脚本 | 用途 |
| :--- | :--- | :--- |
| **解释器** | `p8d.py` | 直接运行 `.8d` 源码，适合快速开发与调试。 |
| **转译器** | `p8dforce.py` | 将源码转译为高性能 C 代码，支持极致的工业吞吐。 |
| **编译器** | `p8cc.py` | 将源码压缩为 `.ejac` 二进制字节码，便于分发。 |
| **虚拟机** | `p8vm.py` | 轻量化运行环境，执行编译后的字节码。 |

---

## 🧬 自举开发 (Self-hosting)

Pink8D 设计之初即为了实现自举。开发者可以使用 P8D 编写自身的编译器：

1. **吸入**：利用 `8[]D ~~8D` 顺序读取源码字符。
2. **鉴别**：利用“压力抵消法”（如 `8....D` 减去 ASCII 特征值）识别形态。
3. **灌注**：利用 `8[=]D 8D~~` 向目标文件轨道压入生成的代码。

## 📦 安装与使用

```bash
    # 使用转译器生成高性能程序
    python3 p8dforce.py script.8d > script.c
    gcc -O3 script.c -o script
    ./script

    # 使用虚拟机运行字节码
    python3 p8cc.py script.8d
    python3 p8vm.py script.ejac
```
