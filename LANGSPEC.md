## 1. 语言概述

### 1.1 设计哲学

Pink8D 基于**形态学**（Morphological）设计原则：

> **"杆长即数值"（Length-as-Value）**
>
> 指令的强度不是通过关键字或符号的数量决定，而是通过特定字符（如 `=`）的数量来表示。

### 1.2 继承关系

Pink8D 是 **Brainfuck** 的艺术化扩展：

| Brainfuck | Pink8D | 创新点 |
|-----------|--------|--------|
| `+` | `8====D` | 形态美学 + 可视化数值 |
| `-` | `8....D` | 使用 `.` 代替减法 |
| `>` | `===>` | 移动方向可视化 |
| `<` | `<===8` | 移动方向可视化 |
| `.` | `8D~` | 输出语义显式化 |
| `,` | `~8D~` | 输入语义显式化 |
| `[` | `8{` | 循环入口 |
| `]` | `}D` | 循环出口 |
| `[>>]` | `8D*` | **新增**：深度解引用 |

### 1.3 技术参数

| 参数 | 值 |
|------|-----|
| 内存模型 | 字节数组（Byte Array）|
| 默认内存大小 | 30,000 字节 |
| p8dforce 内存大小 | 1,000,000 字节 |
| 数值范围 | 0-255（8 位无符号整数）|
| 溢出处理 | 模 256 运算（环形）|
| 指针范围 | 0 至 内存大小-1 |
| 指针越界 | 模运算环绕（Cyclic Wrap）|

## 2. 词法规范

### 2.1 词法分析器

Pink8D 使用**正则表达式**进行形态学词法分析：

```python
pattern = r'~?8[=.]*D[~*]*|[=]*>|<[=]*8|8\{|\}D'
```

### 2.2 Token 类型

| Token 模式 | 类型 | 示例 |
|-----------|------|------|
| `~?8[=.]*D[~*]*` | 运算指令 | `8====D`, `8D~`, `~8D~`, `8D*` |
| `[=]*>` | 右移指令 | `==>`, `===>` |
| `<[=]*8` | 左移指令 | `<==8`, `<===8` |
| `8{` | 循环开始 | `8{` |
| `}D` | 循环结束 | `}D` |

### 2.3 形态学解析规则

#### 2.3.1 运算指令 (`8====D`)

```
语法：~? 8 (= | .)* D ~*

分解：
  - 前缀 ~?    → 输入标记（可选）
  - 核心 8...D → 运算主体
  - 中缀 (= | .)* → 数值计数
  - 后缀 ~*   → 输出标记（可选）或跳转标记（*）
```

**数值计算规则**：
```
value = count('=') - count('.')
```

**语义判定**：
- `~` 前缀存在 → **输入**（IN）
- `~` 后缀存在 → **输出**（OUT）
- `*` 后缀存在 → **跳转**（JMP）
- 否则 → **运算**（ADD if value >= 0, SUB if value < 0）

#### 2.3.2 移动指令

```
右移：  [=]*>      → 指针 += count('=')
左移：  <[=]*8     → 指针 -= count('=')
```

**示例**：
- `===>` → 指针右移 3 位
- `<===8` → 指针左移 3 位

### 2.4 注释规则

Pink8D 遵循 **C++ 风格注释**：

```8d
// 单行注释
/* 多行
   注释 */
```

**注意**：注释不会干扰 Token 匹配，因为正则表达式只识别指令模式。

## 3. 语法规范

### 3.1 程序结构

```
Program     → Instruction*
Instruction → Calc | Move | IO | Loop
Calc        → '~'? '8' ('=' | '.')* 'D' ('~' | '*')?
Move        → ('=')* '>' | '<' ('=')* '8'
IO          → (Calc with ~ or ~ prefix)
Loop        → '8{' Program '}' 'D'
```

### 3.2 语法正确性约束

#### 3.2.1 括号匹配

```
规则：每个 8{ 必须有对应的 }D

错误示例：
8{ 8====D    // 缺少 }D
}D           // 缺少 8{
```

#### 3.2.2 嵌套规则

```
允许嵌套：8{ 8{ ... }D }D
允许跨指令：8{ ===> 8====D }D
```

### 3.3 空白字符处理

- 空格、制表符、换行符**不影响**指令执行
- 建议使用换行和空格增强可读性

## 4. 语义模型

### 4.1 内存模型

```
内存结构：
┌────────────────────────────────────────────────────┐
│  0  │  1  │  2  │ ... │  ptr  │ ... │ 29999      │
├─────┼─────┼─────┼─────┼───────┼─────┼────────────┤
│  0  │  0  │  0  │ ... │ [ptr] │  0  │  0         │
└────────────────────────────────────────────────────┘
     ↑
   数据指针 (DP)
```

**初始状态**：
- 所有内存单元初始化为 0
- 数据指针指向索引 0

### 4.2 操作语义

#### 4.2.1 ADD / SUB（加法/减法）

```
语义：
  m[ptr] = (m[ptr] + value) mod 256

等价于：
  m[ptr] += count('=') - count('.')
  if (m[ptr] < 0) m[ptr] += 256
  if (m[ptr] >= 256) m[ptr] -= 256
```

**示例**：
```8d
8========D    // m[ptr] += 10
8....D        // m[ptr] -= 4
8======D 8==D // m[ptr] += 8
```

#### 4.2.2 RIGHT / LEFT（指针移动）

```
RIGHT:  ptr = (ptr + n) mod MEM_SIZE
LEFT:   ptr = (ptr - n) mod MEM_SIZE
```

**示例**：
```8d
===>           // ptr += 3
<===8          // ptr -= 3
===> ===> ===> // ptr += 9
```

#### 4.2.3 OUT（输出）

```
语义：
  output ASCII(m[ptr])
```

**示例**：
```8d
8D~            // 输出字符（当前值）
8========D 8D~ // 输出 'H' (ASCII 72)
```

#### 4.2.4 IN（输入）

```
语义：
  m[ptr] = input_char
```

**示例**：
```8d
~8D~           // 读取一个字符
```

#### 4.2.5 OPEN / CLOSE（循环）

```
语义：
  while (m[ptr] != 0) {
    // 循环体
  }
```

**循环映射**：
编译器预计算配对位置：
```python
loop_map = {
  open_index: close_index,
  close_index: open_index
}
```

#### 4.2.6 JMP（深度跳转）⭐

```
语义：
  ptr = m[ptr] mod MEM_SIZE
```

**创新点**：
这是 Pink8D 相对于 Brainfuck 的**核心扩展**。

**示例**：
```8d
8D*            // ptr = m[ptr]
```

**用途**：
- 实现链表遍历
- 图的深度优先搜索
- 动态跳转表

## 5. 执行引擎

### 5.1 解释器模型（p8d.py）

```
执行循环：
  while (pc < len(instructions)) {
    fetch();
    decode();
    execute();
    pc++;
  }
```

### 5.2 字节码模型（p8cc.py + p8vm.py）

**Opcode 定义**：
```python
OP = {
    'ADD':    0x01,
    'SUB':    0x02,
    'RIGHT':  0x03,
    'LEFT':   0x04,
    'OPEN':   0x05,
    'CLOSE':  0x06,
    'OUT':    0x07,
    'IN':     0x08,
    'JMP':    0x09
}
```

**字节码格式**：
```python
bytecode = [(opcode, value), ...]
# Example: [(0x01, 10), (0x03, 3), (0x07, 0)]
```

### 5.3 原生转译模型（p8dforce.py）

**C 代码生成**：
```c
unsigned char m[MEM_SIZE] = {0};
int i = 0;

while (m[i]) {
    // 循环体
}
```

**编译优化**：
- 使用 `-O3` 标志
- 内联热点代码
- 消除边界检查（在安全范围内）

## 6. 示例解析

### 6.1 Hello World（B5CRL.8d）

**目标**：输出 "BG5CRL\n"

**分解**：
```8d
// 第一步：'B' (ASCII 66)
8==========D 8==========D 8==========D   // 10 + 10 + 10
8==========D 8==========D 8==========D   // 10 + 10 + 10
8======D 8D~                              // 6, 输出
```

**计算**：
```
(10 + 10 + 10) + (10 + 10 + 10) + 6
= 30 + 30 + 6
= 66
= 'B'
```

### 6.2 简单加法（ok.8d）

**目标**：输出 "OK\n"

**分解**：
```8d
// 第一步：'O' (ASCII 79)
8==========D × 7 + 9 = 79

// 第二步：'K' (79 - 4 = 75)
8....D × 4 = 75

// 第三步：换行 (10)
8..........D × 6 + 4 = 10
```

## 7. 附录

### 7.1 ASCII 速查表

| 字符 | ASCII | 8D 表示 |
|------|-------|---------|
| 0 | 48 | `8========D` × 4 + `8========D` |
| A | 65 | `8==========D` × 6 + `8========D` + `8D` |
| Z | 90 | `8==========D` × 9 |
| a | 97 | `8==========D` × 9 + `8========D` + `8D` |
| z | 122 | `8==========D` × 12 + `8D` |
| \n | 10 | `8==========D` + `8D` |

### 7.2 正则表达式参考

```regex
Token Pattern:
~?8[=.]*D[~*]*|[=]*>|<[=]*8|8\{|\}D

Breakdown:
  ~?8[=.]*D[~*]*   → 运算指令（加/减/IO/跳转）
  [=]*>            → 右移
  <[=]*8           → 左移
  8\{              → 循环开始
  \}D              → 循环结束
```

### 7.3 错误处理

| 错误类型 | 描述 | 处理方式 |
|---------|------|---------|
| 括号不匹配 | `8{` 或 `}D` 未配对 | 抛出 SyntaxError |
| 指针越界 | ptr 超出范围 | 自动模运算环绕 |
| 值溢出 | m[ptr] 超出 0-255 | 自动模 256 |
